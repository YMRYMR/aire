# Security Policy

## Supported Scope

This repository includes a Windows desktop application with:

- provider integrations
- local tool execution
- a loopback local API
- browser automation
- speech and email integrations

Security reports are especially useful for issues involving authentication boundaries, local API behavior, secret handling, unsafe command or browser execution, and data exposure.

## Reporting

Please do not open public issues for unpatched security vulnerabilities.

Instead, report vulnerabilities privately to the project maintainers through the repository security reporting feature once the project is published on GitHub.

If that feature is not yet available, coordinate privately with the maintainer before publishing technical details.

## Disclosure Expectations

- Provide clear reproduction steps
- Identify affected files or features when possible
- Include impact and required preconditions
- Allow reasonable time for remediation before public disclosure
